Données utilisées

Avant de se lancer dans quelques analyses que ce soit, il faut identifier les données que nous pouvons utiliser. Pour notre analyse, nous recours à deux sources Our World in Data et Gapminder.

Our World in Data (OWID)

Le gros du jeu de données avec lequel nous travaillons s’articule avec des données mondiales sur la vaccination. Grâce à une initiative de Mathieu et al. (2021), le site Web de OWID dispose d’une jeu de données très complet sur la vaccination. En plus d’être offertes par pays, par continent et certains États, les données disponibles couvrent plusieurs aspects de la vaccination allant du nombre de doses administrés au personnes ayant reçus une dose de rappels. Autre aspect intéressant de ces données, il est possible de suivre l’évolution quotidienne des différentes variables puisque les données sont disponibles sur une base quotidienne.

Les données peuvent être visualiser et consulter sur le site Web de OWID. La base de données complète est disponibles pour le téléchargement sur GitHub.

Gapminder

Pour compléter notre jeu de données sur la vaccination et être en mesure d’ajouter des nuances à nos analyses, nous avons recours à Gapminder.

Ayant pour objectif de combattre l’ignorance en s’appuyant sur des faits que tous sont mesures de comprendre, Gapminder offre des données fiables qui permettent d’expliquer et de bien comprendre des tendances mondiales.

Plus précisément, nous avons utilisé Gapminder pour récupérer des données en lien avec la population, le PIB et l’espérance de vie. Particularité intéressante de Gapminder, c’est que le site offre les données à la fois pour les pays et pour les continents.

Le site Web de Gapminder peut être consulter à cette adresse : www.gapminder.org

Les jeux de données que nous avons utilisé pour construire son les suivants :

  • Population
  • Produit intérieur brut (PIB)
  • Espérance de vie
  • Géographie

Traitement de données

Téléchargement de données

La première étape de notre travail à consister à mettre en place une stratégie pour le téléchargement des données. Effectivement, comme les données sont récupérés directement sur Internet, il est préférable de récupérer une copie des jeux de données. Ceci répond à deux objectifs, d’abord travailler plus facilement avec une copie locale. De plus, nous permet d’éviter des mauvaises surprises dans l’éventualité où les données viendraient à être retirées du Web.

Nos deux sources de données ont une fréquence de mises à jours différentes. En effet, les données sur la vaccination de OWID sont mises à jour quotidiennement tandis que celles de Gapminder sont plutôt sur une fréquence de mises à jours mensuelles.

Afin de tenir compte de ces particularités, nous avons développer un script qui permet de télécharger quotidiennement ou mensuellement les jeux de données.

Sommairement, voici les étapes qui constitue le script de téléchargement :

  • Mise en place. On part du principe qu’il y a déjà une version des données diponibles localement (qui comprend la date de téléchargement dans son titre).
  • Dernière version. On extrait la date de téléchargement du fichier disponible localement.
  • Comparaison de la date. On compare la date de téléchargement avec la date du jour (ou d’il y a un mois, dans le cas des données mensuelles).
  • Test conditionnel. Si les données disponibles localement ont plus d’une fjournée (ou qu’un mois), on télécharge une nouvelle version des données. On efface ensuite les anciennes données.

Préparation des données

La prochaine étape vers la constitution de notre jeu de données est en lien avec la préparation des différentes données afin de permettre la fusion les différents jeux de données.

Dans l’ensemble des jeux de données, nous nous sommes d’abord assuré que les données sont sous la forme d’un tibble et que chaque variable était dans le format adéquat (ex. facteurs, date, numérique).

Chaque jeu de données étant unique, certains jeux de données ont nécessité des manipulations particulières. Les voici :

  • Données sur la vaccination. Les donées de OWID ont nécessité un nettoyage au niveau des régions géographiques. Après exploration, nous avons constater que certaines juridictions présentaient un code geo commençant par “OWID_.” Nous avons constater deux cas expliquant ceci. D’abord, les continents ont étaient identifiés par ce “OWID_.” L’autre cas référait plutôt à des nations constituantes des Royaume-Uni ou à des groupements supplémentaires (ex. Union européenne, Low middle income, etc.). Dans le premier cas, les codes geo ont été corrigé pour correspondre à ceux des autres données. Alors que dans le deuxième cas, les juridictions ont été retirés des données.
  • Années de vaccination. En considérant que les données sur la vaccination sont quotidiennes et que les autres données sont présentées sur un base annuelle, nous avons ajouter une variable année aux données sur la vaccination.
  • Données sur le PIB. Pour tenir compte de effet d’échelle et offrir une meilleure compréhension des données, le PIB a été ramené en milliards de dollars.

Une fois ces manipulations et ces corrections faites, il a été possible de fusionner tous les jeux de données pour constituer notre jeu de données complet.

Validité des valeurs

Comme mentionné plus haut, notre jeu de données est constitué à partir de deux sources (OWID et Gapminder). Afin de vérifier la validité des données, nous comparons la valeur de la variable total_vaccinations_per_hundred, qui est fourni par OWID, avec une nouvelle variable que nous obtenous à partir de la population de 2021 estimé par Gapminder.

Le jeu de données des données les plus récentes contient 224 observations (des pays ou des continents).

Il faut d’abord noter qu’il n’est pas possible d’effectuer de comparaison pour les 32 observations. Ceci est dû à des valeurs manquantes pour la vaccination ou la population.

Pour 181 des observations, il n’y a pas de différence entre la valeur de la variable total_vaccinations_per_hundred de OWID et celle calculée. Alors que pour 11 observations, il y a une différence entre la valeur de OWID et la valeur calculée.
